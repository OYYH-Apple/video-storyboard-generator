# 视频类型参考

## 7大类视频类型

根据用户描述识别视频类型：

| 类型 | 触发关键词 | 典型场景 | 推荐时长 | 推荐节奏 |
|------|-----------|----------|---------|---------|
| **概念讲解** | 原理、机制、算法、技术、概念 | AI原理、区块链机制、物理定律 | 1-3分钟 | 中速 |
| **书籍介绍** | 书、书籍、读物、推荐、导读 | 书籍推荐、章节导读、读书笔记 | 1-2分钟 | 中速 |
| **热点评论** | 事件、新闻、热点、评论、观点 | 新闻解读、事件分析、观点表达 | 30秒-1分钟 | 快速 |
| **剧情分镜** | 故事、剧情、短片、动画、剧本 | 短片分镜、动画脚本、故事板 | 1-5分钟 | 变化丰富 |
| **产品介绍** | 产品、功能、工具、应用 | 产品发布、功能演示、App介绍 | 30秒-2分钟 | 中速 |
| **数据报告** | 数据、报告、分析、趋势 | 数据可视化、研究报告、趋势分析 | 1-3分钟 | 慢速 |
| **教程指南** | 教程、指南、步骤、操作 | 操作指南、软件教程、流程说明 | 2-5分钟 | 慢速 |

## 类型详解

### 1. 概念讲解类 (Concept Teaching)

**核心目标**: 将抽象概念转化为易懂的视觉表达

**典型结构**:
```
开场(5-10秒): 提出问题/引出概念
↓
核心讲解(60-120秒): 分步骤解析原理
↓
类比/案例(30-60秒): 具体化理解
↓
总结(10-15秒): 回顾要点
```

**推荐配置**:
- 镜头时长: 12-15秒（信息密度高）
- 版式: 专业版（需要详细运镜）
- 主角: 拟人化概念元素/解说员
- 风格: 科技感、简洁现代
- 运镜: 环绕、推近、流程跟拍

**成功要点**:
- 使用类比帮助理解抽象概念
- 分层递进展示复杂结构
- 强调"为什么重要"

---

### 2. 书籍介绍类 (Book Introduction)

**核心目标**: 激发阅读兴趣，传递核心价值

**典型结构**:
```
开场(5-8秒): 书籍封面展示+标题
↓
书籍概览(20-30秒): 作者、背景、定位
↓
核心内容(40-60秒): 3-5个核心观点
↓
价值总结(15-20秒): 适合谁读+收获
↓
行动号召(5-10秒): 购买/阅读引导
```

**推荐配置**:
- 镜头时长: 10-15秒
- 版式: 极简版或专业版均可
- 主角: 书籍封面/关键插图
- 风格: 文艺、知性、与书籍调性一致
- 运镜: 展示书页细节、重点段落特写

**变体类型**:
- **整书概览**: 全面介绍全书框架
- **单章导读**: 聚焦特定章节的深度解读
- **读书笔记**: 个人阅读体验分享

---

### 3. 热点评论类 (Hot Topic Commentary)

**核心目标**: 快速传达观点，引发共鸣

**典型结构**:
```
开场(3-5秒): 事件快速回顾（制造共鸣）
↓
观点陈述(15-25秒): 明确立场
↓
论证展开(30-45秒): 2-3个论据支撑
↓
深度分析(20-30秒): 背后原因/影响
↓
结论(10-15秒): 总结观点+引发思考
```

**推荐配置**:
- 镜头时长: 8-12秒（快节奏）
- 版式: 极简版（快速输出）
- 主角: 评论者形象或新闻素材
- 风格: 严肃、犀利、或轻松调侃
- 运镜: 快切、推近强调、节奏感强

**评论角度**:
- **支持型**: 肯定事件/观点
- **批判型**: 指出问题/风险
- **中立分析型**: 多角度解读

**时效性要求**: 热点类视频需快速产出，建议在事件发酵期（24-72小时内）发布

---

### 4. 剧情分镜类 (Story/Script)

**核心目标**: 讲述引人入胜的故事

**典型结构（三幕式）**:
```
第一幕：铺垫(15-30秒)
  - 建立角色和背景
  - 引入冲突/问题
  
第二幕：发展(60-180秒)
  - 冲突升级
  - 转折/意外
  - 情感高潮
  
第三幕：结局(15-30秒)
  - 冲突解决
  - 主题升华
  - 情感收束
```

**推荐配置**:
- 镜头时长: 灵活（根据情绪变化）
- 版式: 专业版（必须）
- 主角: 故事角色（需要角色提示词）
- 风格: 根据故事类型（悬疑/喜剧/爱情/动作）
- 运镜: 丰富多变，配合情绪曲线

**故事类型**:
| 类型 | 特点 | 镜头风格 |
|------|------|---------|
| 悬疑 | 紧张、反转 | 暗调、快切、特写 |
| 喜剧 | 轻松、幽默 | 明亮、中景、夸张 |
| 爱情 | 温馨、浪漫 | 柔和、近景、慢推 |
| 动作 | 刺激、速度 | 广角、跟拍、快速 |
| 科幻 | 未来、想象 | 冷色调、环绕、特效 |

**特殊要求**:
- 必须生成角色提示词（用于AI生图）
- 需要详细的场景设定
- 考虑目标平台（抖音/YouTube/院线）的画幅要求

---

### 5. 产品介绍类 (Product Introduction)

**核心目标**: 展示产品价值，促成转化

**典型结构**:
```
开场(5-8秒): 产品亮相+核心卖点提炼
↓
痛点共鸣(10-15秒): 用户遇到的问题
↓
解决方案(30-60秒): 产品如何解决（功能演示）
↓
独特优势(20-30秒): 与竞品对比/独特技术
↓
使用场景(15-25秒): 真实应用场景
↓
行动号召(5-10秒): 购买/下载/试用引导
```

**推荐配置**:
- 镜头时长: 10-15秒
- 版式: 专业版（展示细节）
- 主角: 产品本身/产品+用户
- 风格: 与品牌调性一致（科技/时尚/生活）
- 运镜: 360度展示、功能特写、使用跟拍

**变体类型**:
- **新产品发布**: 强调创新和突破
- **功能更新**: 聚焦新增功能
- **产品对比**: 与竞品/旧版对比

**转化优化**:
- 前3秒必须展示产品
- 强调用户收益而非功能
- 使用社会证明（销量/评价）

### 6. 数据报告类 (Data Report)

**核心目标**: 清晰传达数据洞察

**典型结构**:
```
开场(10-15秒): 报告主题+数据来源
↓
关键发现(30-60秒): 3-5个核心数据点
↓
趋势分析(40-80秒): 数据变化趋势+原因解读
↓
对比洞察(30-45秒): 横向/纵向对比
↓
行动建议(20-30秒): 基于数据的建议
```

**推荐配置**:
- 镜头时长: 15-20秒（信息展示需要时间）
- 版式: 专业版（数据可视化）
- 主角: 数据图表/报告人
- 风格: 专业、权威、清晰
- 运镜: 特写数据点、图表动画、数字强调

**数据可视化要点**:
- 使用恰当的图表类型（柱状图/折线图/饼图）
- 突出关键数字（放大/变色/动画）
- 渐进式展示数据（避免信息过载）
- 数据解读 > 数据罗列

---

### 7. 教程指南类 (Tutorial)

**核心目标**: 手把手教会用户操作

**典型结构**:
```
开场(10-15秒): 介绍教程目标和预期成果
↓
准备工作(15-30秒): 所需工具/环境/前提条件
↓
步骤演示(120-300秒): 
  - 步骤1: 操作+解释
  - 步骤2: 操作+解释
  - ...
↓
常见问题(30-45秒): 可能遇到的问题和解决方案
↓
成果展示(15-20秒): 最终效果
↓
拓展建议(10-15秒): 进阶技巧/相关教程
```

**推荐配置**:
- 镜头时长: 15-20秒（操作展示）
- 版式: 专业版（步骤清晰）
- 主角: 操作界面/演示者
- 风格: 清晰、友好、教学感
- 运镜: 界面特写、光标跟随、步骤标注

**教学技巧**:
- 语速放慢，重点重复
- 使用标注（箭头/高亮/文字）
- 每步操作后展示结果
- 预告下一步（"接下来我们..."）

## 类型选择决策树

```
用户想做什么？
│
├─ 解释一个概念/原理 → 概念讲解
│
├─ 介绍一本书 → 书籍介绍
│
├─ 评论事件/表达观点 → 热点评论
│
├─ 讲故事 → 剧情分镜
│
├─ 推广产品 → 产品介绍
│
├─ 展示数据/报告 → 数据报告
│
└─ 教别人怎么做 → 教程指南
```
