# 镜头时长参考

## 镜头时长规则

**核心规则：单个镜头时长不得超过用户设定的上限（默认15秒）**

时长上限可在头脑风暴阶段自定义，以适应不同视频类型和内容需求：
- **15秒以下**：保持紧凑节奏，适合快节奏短视频
- **15-20秒**：适合需要更多信息密度的场景（教学、数据展示）
- **20秒以上**：仅在特殊叙事需求时使用

详见 `questions-by-type.md` 中的"镜头时长限制"问题。

## 短视频优化策略（重要）

**针对总时长≤60秒的视频，采用动态镜头规划而非机械拆分：**

### 短视频镜头数量建议

| 视频总时长 | 建议镜头数 | 单镜头平均时长 | 开场镜头建议 |
|-----------|-----------|---------------|-------------|
| 15秒 | 2-3个 | 5-7秒 | 3秒 |
| 30秒 | 3-5个 | 6-10秒 | 3-5秒 |
| 45秒 | 4-6个 | 7-11秒 | 4-5秒 |
| 60秒 | 5-8个 | 7-12秒 | 4-5秒 |

### 短视频镜头分配原则

1. **开场镜头（前3-5秒）**：必须抓住注意力，节奏较快
2. **核心内容镜头**：承载主要信息，时长适中（8-12秒）
3. **结尾镜头**：收束和CTA，节奏平稳（5-8秒）

### 短视频 vs 标准视频策略对比

| 特征 | 短视频（≤60秒） | 标准视频（>60秒） |
|------|----------------|------------------|
| 镜头时长上限 | 12秒（而非15秒） | 15秒（默认） |
| 镜头密度 | 更密集，节奏更快 | 适中节奏 |
| 拆分逻辑 | 按镜头数量规划，避免过度拆分 | 按时长规则拆分 |
| 运镜变化 | 每个镜头都应有变化 | 可有轻微重复 |
| 最短镜头 | 不低于3秒 | 不低于2秒 |

### 短视频拆分示例

**30秒视频示例：**
```
镜头1（开场）：4秒 - 吸引注意力，引出主题
镜头2（核心）：10秒 - 主体内容A部分
镜头3（核心）：10秒 - 主体内容B部分
镜头4（结尾）：6秒 - 总结/CTA
---
总计：30秒，4个镜头（节奏适中，不过碎）
```

**60秒视频示例：**
```
镜头1（开场）：5秒 - 引入话题
镜头2（核心）：12秒 - 要点A详解
镜头3（核心）：12秒 - 要点B详解
镜头4（核心）：12秒 - 要点C详解
镜头5（过渡）：7秒 - 汇总过渡
镜头6（结尾）：7秒 - 总结/CTA
---
总计：55秒，6个镜头（留5秒弹性）
```

## 时间拆分原则（标准视频 >60秒）

根据用户设定的时长上限（默认15秒）进行拆分：

| 原镜头时长 | 拆分建议（默认上限15秒） |
|-----------|-------------------------|
| 15秒以内 | 保持单个镜头 |
| 16-30秒 | 拆分为2个镜头（约8-15秒/个） |
| 31-45秒 | 拆分为3个镜头（约10-15秒/个） |
| 45秒以上 | 拆分为N个镜头（每个≤15秒） |

**自定义上限时的拆分建议：**
- 如果设定上限为20秒：16-40秒拆为2个，41-60秒拆为3个
- 如果设定上限为10秒：11-20秒拆为2个，21-30秒拆为3个

拆分原则：确保每个镜头时长不超过用户设定的上限，且拆分后的镜头时长尽可能接近上限。

## 按视频类型的镜头时长建议

| 视频类型 | 建议单镜头时长 | 说明 |
|---------|---------------|------|
| 快节奏短视频 | 3-8秒 | 抖音/快手风格 |
| 教学讲解 | 8-12秒 | 保持清晰节奏 |
| 剧情分镜 | 5-15秒 | 根据情绪调整 |
| 数据展示 | 10-15秒 | 数据需要时间阅读 |
| 产品演示 | 8-12秒 | 突出关键功能 |

## 拆分后的运镜变化

拆分后的相邻镜头必须有明显的运镜变化，避免视觉重复。

详细运镜类型见 `references/camera-movements.md`。
